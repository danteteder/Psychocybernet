## Decision Frameworks

### The 25% Rule
**For any revenue-generating activity:**
- If someone else can do it for ≤25% of revenue, delegate it
- Example: Sales reps get 25%, you keep 75% for delivery
- Applies to: Sales, content creation, customer support

### The 24/7 Test
**Before building anything, ask:**
- Will this generate revenue while I sleep?
- Can it run without my direct involvement?
- Does it scale linearly or exponentially?

If no to any: Redesign or don't build.

### The Leverage Multiplier
**Rank activities by leverage:**
1. **Code**: Writes once, runs forever (highest)
2. **Content**: Creates once, sells forever
3. **Systems**: Automates recurring work
4. **People**: Delegates hourly work (lowest)

Always work on #1 before #2, #2 before #3, etc.

## Mental Models

### Compound Interest of Effort
Small daily improvements in systems compound exponentially:
- 1% better automation daily = 37x improvement in 1 year
- Focus on systems, not goals

### Barbell Strategy
**Safe side (80%):**
- Proven revenue streams (consulting, SaaS)
- Low risk, predictable income

**Risky side (20%):**
- Experimental projects (new products, markets)
- High risk, asymmetric upside

Avoid the middle (medium risk, medium reward).

### Optionality
Build systems that create future options:
- Email list → launch anything
- Audience → monetize multiple ways
- Code library → ship faster next time
